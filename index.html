Formularios

Ideas principales

  • Los formularios en HTML son una forma de recoger información de los usuarios en una página web. Se crean usando la etiqueta (form), y dentro de ella se colocan diferentes tipos de campos como cajas de texto, casillas de verificación, botones de opción , menús desplegables, y áreas de texto grandes.
  • Los formularios se envían al servidor con los métodos GET o POST, que se definen con los atributos action (la dirección a donde se envían los datos) y method (cómo se envían los datos).
  • El posicionamiento relativo desplaza un elemento respecto de su posición original establecida mediante el posicionamiento normal.
  • HTML también ofrece formas de validar los datos, como hacer que ciertos campos sean obligatorios (required) o que sigan un formato específico (pattern). Para mejorar la accesibilidad, es importante usar etiquetas (label) que describan los campos. Además, los formularios pueden ser organizados en grupos con (fieldset y legend).
  • Los formularios pueden ser personalizados con CSS y validados o enviados sin recargar la página usando JavaScript o AJAX.

Apuntes de la clase

  • Un formulario en HTML es una sección dentro de una página web que permite a los usuarios enviar datos al servidor para realizar diversas acciones, como registros, inicio de sesión, encuestas, entre otros.
  • Los formularios utilizan la etiqueta
    , y dentro de ellos se incluyen distintos campos de entrada (inputs), botones y otros elementos interactivos.

Resumen

En resumen, los formularios en HTML permiten recoger información de los usuarios, usando campos como cajas de texto, botones y menús. Se envían al servidor con los métodos GET o POST. HTML ofrece opciones para validar datos, hacer campos obligatorios y mejorar la accesibilidad. También se pueden personalizar con CSS y hacer más interactivos con JavaScript. Volver Arriba

Tablas

Ideas principales

  • Estructura básica: Se crean con la etiqueta (table) y se organizan en filas (tr) y celdas (td para datos y th para encabezados).
  • Encabezados y datos: Los encabezados de las columnas se definen con th y se muestran generalmente en negrita y centrados, mientras que los datos se colocan en las celdas td.
  • Agrupación: Las tablas pueden tener secciones como encabezado (thead), cuerpo (tbody) y pie (tfoot) para organizar mejor la información.
  • Estilos: Se pueden personalizar con CSS para mejorar su apariencia, como agregar bordes, colores o espaciado.
  • Accesibilidad: Las tablas pueden mejorarse para ser más accesibles mediante atributos como scope en los encabezados, lo que ayuda a los usuarios con discapacidad visual.

Apuntes de la clase

  • El mejor para utilizar es el display inline-block, permite controlar ancho y alto, además respeta paddings y margin.
  • Se puede usar el display none, para hacer elementos desplegables que no ocupan espacio hasta que el usuario lleva acabo alguna acción.
  • El display table permite insertar tablas a los elementos anidados para organizar información.

Resumen

En resumen, las tablas en HTML organizan datos en filas y columnas usando las etiquetas (table), (tr): (filas), (td): (celdas de datos) y (th): (encabezados). También se pueden agrupar las filas en partes como encabezado, cuerpo y pie con (thead), (tbody) y (tfoot). Además, se pueden estilizar con CSS para mejorar su apariencia. Volver Arriba

Especificidad

Ideas principales

  • Si hay varias reglas que afectan a un mismo elemento, la regla más específica (con más detalles) tendrá prioridad.
  • La especificidad se mide con una puntuación basada en el tipo de selectores usados:
    ID (ej. #mi-id): Alta especificidad.
    Clases, atributos y pseudoclases (ej. .mi-clase, [type="text"], :hover): Moderada especificidad.
    Elementos y pseudoelementos (ej. div, p, ::before): Baja especificidad.
  • Se asigna un valor numérico para cada tipo de selector:
    ID: 100 puntos
    Clase/atributo/pseudoclase: 10 puntos
    Elemento/pseudoelemento: 1 punto El valor total de la especificidad se calcula sumando los puntos.
  • Si dos reglas tienen la misma especificidad, la última regla definida en el CSS tendrá prioridad (por el principio de cascada).
  • El uso de !important puede forzar la aplicación de una regla, pero debe usarse con cuidado, ya que puede complicar la gestión de estilos y la especificidad.

Apuntes de la clase

  • Hay que tener en cuenta el orden de los atributos a modificar en los gradient.
  • Se añaden nuevas funciones que tal vez no todos los navegadores soporten, tal vez tiene que tener un plan b en caso de que no sea soportado.
  • Se tiene que tener en cuenta todos los factores para usar correctamente las nuevas propiedades.

Resumen

En resumen, la especificidad en CSS determina qué reglas de estilo se aplican a un elemento cuando hay varias que coinciden. Se calcula según el tipo de selectores: ID tiene la mayor especificidad (valor 100). Clases tienen una especificidad media (valor 10). Elementos tienen la menor especificidad (valor 1). Si varias reglas tienen la misma especificidad, se aplica la última definida (por la cascada). El uso de !important puede forzar que una regla se aplique sobre otras, pero debe usarse con moderación.Volver Arriba

After y Before

Ideas principales

  • Los pseudo-elementos ::before y ::after en CSS se utilizan para insertar contenido antes o después del contenido de un elemento HTML, sin modificar el HTML directamente.
  • Ambos requieren la propiedad content para funcionar y se emplean comúnmente para agregar decoraciones visuales, como comillas, iconos o efectos sin alterar la estructura del documento.
  • ; ::before inserta contenido antes del contenido del elemento, mientras que ::after lo hace después.
  • Son útiles para mejorar el diseño y la presentación de una página, permitiendo insertar elementos decorativos o funcionales de forma más flexible y separada de la lógica HTML.

Apuntes de la clase

  • Los pseudo-elementos son elementos virtuales que se crean mediante CSS.
  • Permiten insertar contenido antes (::before) o después (::after) del contenido real de un elemento sin modificar el HTML.
  • La propiedad content es obligatoria para que los pseudo-elementos funcionen. Especifica el contenido que se inserta.

Resumen

En resumen, los pseudo-elementos ::before y ::after permiten insertar contenido visual antes o después de un elemento HTML, utilizando solo CSS, lo que mejora la flexibilidad del diseño sin tocar el código HTML directamente.Volver Arriba

Colores

Ideas principales

En HTML y CSS, los colores se pueden definir de varias maneras:

  • Nombres de colores: Usando palabras predefinidas como red, blue, o green.
  • Hexadecimales: Con códigos como #FF5733 para representar colores en formato RGB (rojo, verde, azul).
  • RGB: Especificando colores con valores numéricos, por ejemplo, rgb(255, 0, 0) para rojo.
  • RGBA: Similar a RGB, pero con un valor adicional para la transparencia, como rgba(255, 0, 0, 0.5).
  • HSL: Usando Tono, Saturación y Luminosidad, por ejemplo, hsl(120, 100%, 50%) para verde.
  • HSLA: Igual que HSL, pero con transparencia (hsla(120, 100%, 50%, 0.5)).
  • transparent: Para definir un color completamente transparente.

Apuntes de la clase

  • Los colores se definen utilizando CSS (Cascading Style Sheets). Existen varias formas de especificar los colores para texto, fondo y otros elementos visuales.
  • Los colores se pueden especificar usando palabras clave predefinidas como: red, blue, green, black, white, yellow, etc.
  • Los colores se definen usando un código de 6 dígitos, representando los valores en formato RGB (Rojo, Verde, Azul).
  • La sintaxis es #RRGGBB, donde RR, GG, y BB son los valores de rojo, verde y azul en formato hexadecimal (de 00 a FF).
  • El modelo RGB usa tres valores numéricos entre 0 y 255 para representar el nivel de rojo, verde y azul.

Resumen

En resumen, en HTML y CSS, se pueden usar diferentes formas para definir colores: con nombres de colores como red o blue, códigos hexadecimales como #FF5733, valores RGB (por ejemplo, rgb(255, 0, 0)), RGBA para agregar transparencia (rgba(255, 0, 0, 0.5)), HSL (como hsl(120, 100%, 50%)), HSLA con transparencia, y transparent para hacer un color totalmente invisible. Estas opciones te permiten controlar el color y la transparencia de los elementos en una página web.Volver Arriba

Header

Ideas principales

  • El elemento (header) en HTML se utiliza para definir la cabecera de una página web o de una sección específica, y suele contener elementos como el logo, el título del sitio, y el menú de navegación.
  • Su propósito es proporcionar una estructura clara y semántica al contenido, mejorando la accesibilidad y la comprensión del documento tanto para los usuarios como para los motores de búsqueda.
  • Además, el (header) puede usarse varias veces en una página para marcar las cabeceras de diferentes secciones. Este elemento es clave para organizar la presentación de un sitio web de manera lógica y accesible.

Apuntes de la clase

  • El (header) es un elemento semántico de HTML utilizado para definir la cabecera de una página web o de una sección específica.
  • Ayuda a organizar y estructurar el contenido, especialmente al inicio de la página o en cada sección del sitio.
  • Se utiliza para contener información introductoria o de navegación.
  • Mejora la accesibilidad y la SEO (optimización para motores de búsqueda) porque facilita que los buscadores entiendan la estructura de la página.

Resumen

En resumen, el elemento (header) en HTML se utiliza para definir la cabecera de una página o sección. Contiene información clave como el logo, el título, y el menú de navegación. Su uso mejora la estructura semántica de la página, facilita la accesibilidad para usuarios y motores de búsqueda, y ayuda en la organización del contenido. Aunque generalmente se coloca al principio de una página, también puede repetirse en diferentes secciones. El (header) es esencial para una mejor comprensión del contenido y para optimizar la experiencia del usuario en la web. Volver Arriba

Footer

Ideas principales

  • El elemento (footer) en HTML se utiliza para definir la parte inferior de una página o sección, y generalmente contiene información complementaria como derechos de autor, enlaces a políticas de privacidad, términos de servicio, redes sociales e información de contacto.
  • Normalmente se coloca al final del documento, pero también puede aparecer al pie de cada sección o artículo. Su uso mejora la estructura semántica del contenido, facilitando la navegación y mejorando la accesibilidad, ya que permite a los usuarios y motores de búsqueda identificar fácilmente la información adicional.
  • Además, el (footer) no debe confundirse con el (header), que se usa para la parte superior de la página.

Apuntes de la clase

  • El elemento (footer) se usa para definir la parte inferior de una página web o una sección específica del contenido.
  • Contiene información complementaria, como derechos de autor, enlaces a políticas, redes sociales, y otros detalles importantes.
  • Su propósito es mostrar información no esencial para la navegación principal, pero importante para el usuario
  • Se puede utilizar dentro de secciones o artículos para marcar el pie de cada uno de ellos.

Resumen

En resumen, el elemento (footer) en HTML se usa para definir la parte inferior de una página o sección, y contiene información complementaria como derechos de autor, enlaces a políticas, redes sociales e información de contacto. Generalmente se coloca al final del documento, pero también puede aparecer al pie de cada sección. Mejora la estructura semántica, facilita la navegación y mejora la accesibilidad, ayudando a los motores de búsqueda a identificar fácilmente la información adicional. Volver Arriba
